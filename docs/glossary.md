# PAM Glossary

## Memory

A durable Markdown record stored in `.ai-memory/`. Active memories are eligible
for LLM context. Proposed memories are assisted-mode review items; archived,
deleted, and noise memories are kept out of normal context.

## Proposed Memory

A memory created in assisted capture mode that needs user approval before it
becomes active. Review proposed memories with `pam review` or the UI Evidence
view.

## Observation

A short lifecycle event recorded by hooks, such as session start, prompt event,
or checkpoint attempt. Observations are not the same as durable memories and are
subject to redaction and retention.

## Recommendation

A maintenance suggestion generated by the intelligence layer. A recommendation
is not a memory. It may suggest archiving, marking noise, resolving a
contradiction, or creating a distilled memory.

## Distillation

A synthetic memory created from several source memories. Source IDs are
preserved so the user can inspect where the summary came from.

## Context Preview

The exact block of memory text PAM would provide to an LLM for a query. This is
the best way to verify what the agent can actually read.

## Knowledge Graph

A preview of entities and relations inferred from memories. It is inspectable
and evidence-backed, not a hidden autonomous memory store.

## Concept

A normalized semantic theme used to group memories in the UI, recommendations,
and context explanations. It should come from memory content or client-provided
semantic labels. Capture metadata such as agent names, checkpoint markers, model
names, and memory types are facets, not strong concepts.

## Noise

Memory that should not influence context because it is low-signal, temporary, or
irrelevant. Noise is hidden by default but remains restorable.

## Supersession

The link between an older memory and a newer memory that replaces or corrects
it. Supersession archives the older memory instead of overwriting history.

## Backup

A `.bak` copy written under `.ai-memory/backups/` before physical deletion. Use
`pam restore <id>` to restore the latest backup when the original Markdown
file has been removed.

## Assisted Capture

An optional review mode. Agents can propose memories through MCP tools or
generated hooks, but the user approves them before they become active.

## Manual Capture

A mode where durable memories are only created by explicit user commands such as
`pam add`.

## Auto Capture

The default mode. Agent-created memories become active without review after
automatic consolidation and contradiction handling.
